# OverCR v0.4.2 — Reference Documentation

## Overview

OverCR v0.4.2 adds live provider-backed inference through the Hermes CLI adapter.

### What's New

- **HermesCLIAdapter**: Production inference adapter that calls `hermes chat -q` via subprocess
- **Runtime-Agnostic**: OverCR doesn't hold API keys or implement HTTP client logic
- **Dynamic Provider Routing**: All model routing goes through Hermes config
- **Fallback to Deterministic**: Inference failures don't advance task state

### Architecture

```
OverCR Substrate (no API keys)
    |
    +---> HermesCLIAdapter (subprocess)
              |
              +---> Hermes CLI (runtime-agnostic router)
                        |
                        +---> Provider (Ollama, OpenAI, etc.)
```

## Files Changed

- `runtime/hermes_inference_adapter.py`: New v0.4.2 adapter implementation

## Components

### HermesCLIAdapter

Live inference adapter that calls the Hermes CLI for real model inference.

**Key Properties:**

- `adapter_type`: `"hermes_cli"`
- `is_available()`: Checks Hermes CLI availability (cached after first check)
- `invoke(prompt, config)`: Makes actual inference call via `hermes chat -q`
- `dry_run(config)`: Validates availability without making a call
- `_resolve_cli_path()`: Resolves `HERMES_CLI_PATH` or `hermes` in PATH

**Adapter Configuration:**

```yaml
# inference_routing.yaml snippet
production:
  adapter: "hermes_cli"
  model: "glm-5.1:cloud"
  provider: "ollama-cloud"
  timeout_s: 45
  require_available: true
  fallback_to_deterministic: true
```

### Domain Configuration

Domains can enable inference independently:

```yaml
_knower:
  claim_review:
    enabled: true
    adapter: "hermes_cli"  # Options: mock, hermes, hermes_cli
    model: "glm-5.1:cloud"
    fallback_model: "qwen3-coder-next"
    provider: "ollama-cloud"
    timeout_s: 45
    max_retries: 1
    fallback_to_deterministic: true
    prompt_template: "subagents/knower/inference_prompt.md"

myth_fact:
  enabled: true
  adapter: "hermes_cli"
  model: "glm-5.1:cloud"
  fallback_model: "qwen3-coder-next"
  provider: "ollama-cloud"
  timeout_s: 45
  max_retries: 1
  fallback_to_deterministic: true
```

### Inference Result Types

All inference results are wrapped in `InferenceResult`:

```python
@dataclass
class InferenceResult:
    metadata: InferenceMetadata
    packet: Optional[dict]          # Model-produced packet (untrusted)
    fallback_packet: Optional[dict]  # Deterministic fallback packet
```

**Status Enum:**

- `InferenceStatus.SUCCESS`: Inference succeeded, packet available
- `InferenceStatus.FALLBACK_USED`: Deterministic worker used instead
- `InferenceStatus.TIMEOUT`: Model call timed out
- `InferenceStatus.ERROR`: Model call failed (auth, network, etc.)
- `InferenceStatus.MALFORMED_OUTPUT`: Output unparseable
- `InferenceStatus.VALIDATION_FAILED`: Packet failed L1-L6 validation

### Programmatic Usage

```python
from runtime.inference_adapter import get_adapter

# Get adapter
adapter = get_adapter("hermes_cli")

# Check availability
if adapter.is_available():
    print("Hermes CLI is available")
else:
    print("Hermes CLI not available")

# Dry run (availability check without inference)
config = {
    "domain": "claim_review",
    "model": "glm-5.1:cloud",
    "provider": "ollama-cloud",
    "timeout_s": 30,
    "task_id": "task-demo",
    "input_context": {...}
}

availability = adapter.dry_run(config)
print(f"Available: {availability['available']}")
if not availability['available']:
    print(f"Error: {availability['error']}")

# Make actual inference call
result = adapter.invoke(prompt, config)

if result.success:
    print(f"Adapter: {result.metadata.adapter_type}")
    print(f"Model: {result.metadata.selected_model}")
    print(f"Packet keys: {list(result.packet.keys())}")
    print(f"Inference source: {result.packet.get('_inference_source')}")
else:
    print(f"Error: {result.metadata.error_message}")
```

### Environment Variables

- `HERMES_CLI_PATH`: Path to hermes binary (overrides PATH lookup)
- `OLLAMA_API_KEY`: For legacy `hermes` adapter (not needed for v0.4.2)

## Governance Rules (Enforced Regardless of Adapter)

1. **Model output CANNOT change doctrine**
2. **Model output CANNOT bypass approval gates**
3. **Model output CANNOT route directly to another subagent**
4. **Model output CANNOT claim live browsing occurred**
5. **Model output is UNTRUSTED until validated through 6-level validator**
6. **On inference failure, task MUST NOT advance state**
7. **Deterministic fallback MUST always be available**

## Configuration Priority

Adapter type configuration is resolved in this order:

1. Domain config (`inference_routing.yaml`): `claim_review.adapter: "hermes_cli"`
2. Default (`_inference_defaults`): `adapter: "mock"`
3. Factory default: `"mock"`
