## Implementation Highlights

### 1. Runtime-Agnostic Design

OverCR v0.4.2坚持 its runtime-agnostic principle:
- **No API keys**: All authentication delegated to Hermes
- **No HTTP client**: Uses subprocess boundary, not direct HTTP
- **Model routing via Hermes**: Swapping providers in Hermes config changes behavior without touching OverCR

### 2. Governance Enforcement

All inference outputs are treated as UNTRUSTED until validated:
```yaml
_governance:
  model_output_cannot_change_doctrine: true
  model_output_cannot_bypass_approval_gates: true
  model_output_cannot_route_directly: true
  model_output_cannot_claim_browsing: true
  model_output_untrusted_until_validated: true
  inference_failure_must_not_advance_state: true
  deterministic_fallback_always_available: true
```

### 3. Safe Fallback Pattern

On inference failure, the system falls back to deterministic (template) workers:
```yaml
production:
  fallback_to_deterministic: true
  require_available: true
```

This ensures the system continues to function even if Hermes is unavailable or inference times out.

### 4. Audit Trail

Each inference result carries full metadata:
```python
metadata = InferenceMetadata(
    inference_attempt_id="inf-claim_review-task-001-2026051100000000",
    domain="claim_review",
    adapter_type="hermes_cli",
    selected_model="glm-5.1:cloud",
    selected_provider="ollama-cloud",
    elapsed_s=41.4,
    status=InferenceStatus.SUCCESS,
    raw_output_summary="[truncated to 500 chars]",
)
```

### 5. Factory Pattern

Adapter selection via factory function:
```python
def get_adapter(adapter_type: str, **kwargs) -> BaseInferenceAdapter:
    if adapter_type == "mock":
        return MockInferenceAdapter()
    elif adapter_type == "hermes":
        return HermesInferenceAdapter(**kwargs)  # Legacy
    elif adapter_type == "hermes_cli":
        return HermesCLIAdapter(**kwargs)  # v0.4.2 production
```

1. **Live inference is slow**: ~40-50s for cloud model. Use mock adapter for development.
2. **JSON output best-effort**: Model may return raw text, not parseable JSON
3. **Timeout handling**: Models may exceed timeout, requiring longer `timeout_s` values

## Recommendations

1. **For development**: Use mock adapter for instant iteration
2. **For CI/CD**: Mock adapter (deterministic, no dependency on external model)
3. **For production**: HermesCLIAdapter with proper timeout configuration
4. **Testing workflow**: Test inference pipeline with mock, validate with Hermes CLI
