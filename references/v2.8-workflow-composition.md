# OverCR v2.8.0 — Workflow Composition & Conditional Routing

## Release Summary

v2.8.0 extends the v2.3 linear DAG workflow system with governed conditional orchestration: branching, retry, escalation, subworkflow composition, validated state machines, and append-only branch tracing.

This is **controlled workflow intelligence** — not autonomous agent swarms.

## Architecture Decisions

### 1. Data-Driven Conditions (No eval())
`ConditionEvaluator` evaluates conditions through deterministic comparisons. Supported: confidence thresholds, trust tiers, validation results, approval decisions, contradiction presence, retry exhaustion, escalation triggers, and fallback activation. Condition types are fixed enums, not arbitrary expressions.

### 2. Explicit Routing with Audit
`RoutingPolicy` determines next_node via: conditional edges (evaluated by priority) → static edges → fallback → escalation → rejection. Every decision records: source/target, decision type, matched condition, evaluated conditions, state snapshot, operator, and timestamp.

### 3. Bounded Retries
`RetryPolicy` caps at `max_retries` (0-10). Retries only trigger on matching conditions. After exhaustion, optional escalation fires. Deterministic fallback triggers at `fallback_threshold`. Every attempt is recorded.

### 4. Escalation Targets
Five escalation targets: `approval_queue`, `operator_review`, `contradiction_review`, `governance_review`, `sandbox_review`. Escalation records: escalation_id, target, severity, pause_workflow flag, resolution state.

### 5. Version-Pinned Subworkflows
`SubworkflowLoader` loads subworkflows with exact version matching. Cycle detection via Kahn's algorithm. Input/output mapping between parent and child namespaces. Audit linkage via parent_run_id / child_run_id.

### 6. Validated State Machine
10 states with explicit legal transitions. `InvalidTransitionError` on illegal moves. All valid transitions recorded in append-only history.

### 7. Branch Trace — The Audit Backbone
`BranchTrace` records every decision: conditional branches, retries, escalations, fallbacks, static routing, subworkflow calls. Entries are serializable and replayable.

## Governance Boundaries

1. No hidden branching — every branch recorded in BranchTrace
2. No autonomous recursive workflows — cycle detection in SubworkflowLoader
3. No self-editing workflow graphs — templates are immutable after load
4. No silent retries — every attempt audited
5. No branch execution without audit trace
6. No escalation bypass — must go through EscalationPolicy
7. No subworkflow execution without version pinning
8. No runtime mutation of frozen workflows
9. No dynamic eval() — conditions are data-driven comparisons only
