# OverCR v2.9.0 — Release Readiness

## Release Candidate Criteria

A v2.9.0 release candidate is ready when ALL of the following pass:

### Structural Integrity
- [ ] All required directories exist (integration/system_validator check)
- [ ] All required files exist
- [ ] All schemas are present and parseable
- [ ] All workflow templates pass validation
- [ ] All frozen workflows are intact

### Runtime Compatibility
- [ ] at least one sandbox backend available (local always passes)
- [ ] Bubblewrap binary found OR warning accepted
- [ ] Firejail binary found OR warning accepted
- [ ] Hermes CLI adapter available OR mock adapter sufficient

### Replay Integrity
- [ ] All workflow templates replay deterministically (same input = same node order)
- [ ] Audit entry types preserved across runs
- [ ] Branch trace ConditionEvaluator is deterministic
- [ ] State machine rejects illegal transitions

### Release Cleanliness
- [ ] No __pycache__ directories with content
- [ ] No .pyc bytecode files in source tree
- [ ] No transient runtime debris
- [ ] .gitignore covers all common patterns
- [ ] No files > 1MB in source tree

### Cross-System Consistency
- [ ] Task references are valid (states, subagents)
- [ ] Workflow edge→node references are consistent
- [ ] Memory records are parseable
- [ ] Provenance tracker module loads
- [ ] Execution receipt round-trip works
- [ ] Escalation records resolve correctly

### Migration Safety
- [ ] All v1 components preserved
- [ ] validate_packet.py signature intact
- [ ] Schema versions are pinned
- [ ] Workflow templates have version pins
- [ ] v2.3→v2.8 upgrade chain is intact

### Recovery Capability
- [ ] Cold-start reconstruction succeeds
- [ ] Audit traces reconstruct correctly
- [ ] Partial artifact loss is detected (not ignored)
- [ ] Rollback snapshots save and restore

### Full Test Suite
- [ ] All 28 regression tests pass (run_all.py)
- [ ] All integration tests pass
- [ ] No test regressions from any subsystem

## Release Artifacts

The v2.9.0 release candidate should include:
1. All source files (the git repository)
2. Reference documentation (references/v2.9-*)
3. Integration validation scripts (scripts/check_v2_integrity.py, etc.)
4. Integration test suite
5. Compatibility report (machine-readable JSON)
6. Release readiness checklist (this document)

## Blockers for v2.10.0 Stable Release

### Resolved by v2.9.0
- Integration integrity validation
- Replay determinism verification
- Release cleanliness checking
- Cross-system reference consistency
- Cold-start and recovery simulation

### Still Unverified (remain as known risks)
- Long-running workflow behavior (>5 min execution)
- Concurrent workflow execution (parallel trace files)
- Large trace file performance (>100K audit entries)
- Network-isolated recovery (air-gapped restoration)
- Full v1→v2 state migration (automatic conversion)
