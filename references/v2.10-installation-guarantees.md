# OverCR v2.10.0 — Installation Guarantees

## Clean Extraction

- Release archive extracts to a self-contained `overcr/` directory
- Zero hardcoded absolute paths (validated by InstallValidator)
- Zero __pycache__ or .pyc files in the archive
- Zero runtime state (audit logs, traces, receipts, snapshots) in the archive

## Dependency Requirements

### Required (stdlib only)
Python >= 3.10. No pip packages required for core operation.
All validation, governance, and replay work with stdlib.

### Optional
| Package    | Purpose                     | Graceful Degradation |
|------------|-----------------------------|---------------------|
| yaml       | Config file parsing         | Falls back to JSON   |
| requests   | Web ingestion fetches       | Mock fetcher in test |
| rich       | TUI terminal rendering      | Plain text fallback  |
| markdown   | Document ingestion          | Text-only ingestion  |
| pydantic   | Type validation enhancement | Dataclass validation |

## Bootstrap Procedure

1. Extract archive: `tar xzf overcr-2.10.0.tar.gz`
2. Enter directory: `cd overcr`
3. Verify integrity: `python3 scripts/check_v2_integrity.py`
4. Run tests: `python3 tests/run_all.py`

## Platform Targets

| Platform         | Status              | Notes                          |
|------------------|---------------------|--------------------------------|
| Linux x86_64     | Full                | All backends available         |
| Linux aarch64    | Full                | All backends available         |
| WSL2             | Full                | Local backend, bubblewrap may need kernel config |
| macOS x86_64     | Local backend only  | bubblewrap/firejail unavailable |
| macOS arm64      | Local backend only  | bubblewrap/firejail unavailable |

## Filesystem Requirements

- POSIX file permissions (chmod works)
- Symlink support
- Case-sensitive preferred (ext4, APFS case-sensitive)
- At least 100MB free for runtime state
