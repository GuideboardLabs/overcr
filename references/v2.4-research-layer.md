# OverCR v2.4.0 — Research & Knowledge Layer

## Release Summary

v2.4.0 expands KnowER from a single worker into a governed research and knowledge subsystem. It introduces document ingestion, source registration with trust tiers, provenance tracking across every transformation, contradiction detection that never auto-resolves, and structured research packet generation with full citation chains.

This is about **trusted knowledge operations** — not unrestricted web crawling, not autonomous intelligence gathering, not RAG pipelines. Every output is attributable, auditable, replayable, provenance-aware, and operator-reviewable.

## Architecture Decisions

### 1. Source Records as Immutable JSON Artifacts
Every source is a JSON record in `knowledge/sources/`. Once registered, a source's content is hashed (SHA-256) and its record is append-only. Duplicate content is detected by hash. Sources go stale but are **never deleted**.

### 2. Trust Tiers Are Operator-Controlled
Five trust tiers exist: `verified`, `reputable`, `unknown`, `suspicious`, `rejected`. Trust tiers never auto-escalate. Only `set_trust_level()` with an explicit operator identity can change a tier. This prevents model output from silently elevating source authority.

### 3. Markdown-First Ingestion
The `DocumentIngestor` supports markdown (with YAML frontmatter metadata extraction), JSON (parse + re-normalize), and plain text. No PDFs yet (future extension). No OCR. No embeddings. Content is normalized and stored in `knowledge/documents/`.

### 4. Deterministic Indexing
`KnowledgeIndex` builds keyword and tag indexes from source records using regex word extraction. No vector DB, no embeddings, no ML. The index is fully rebuildable from source records at any time. Index files are JSON on disk.

### 5. Provenance as Append-Only JSONL
`ProvenanceTracker` writes tracker entries as JSONL files (one per source). Every origin recording, ingestion method, transformation, workflow usage, citation, and contradiction is appended. Entries are never rewritten or deleted. This is the foundation of audit integrity.

### 6. Contradictions Are Flagged, Never Resolved
`ContradictionDetector` uses pattern-based comparison to detect conflicts: direct negations, opposing polarity keywords with shared subjects. Both sides of every contradiction are preserved. The operator must decide — acknowledge, investigate, or discard. No automatic compromise is synthesized.

### 7. Research Packets With Full Provenance
`ResearchPacketBuilder` produces four packet types: claim review, research brief, contradiction summary, source quality. Every packet carries `provenance_chain`, `citation_refs`, `confidence_scoring`, `contradiction_indicators`, and `audit_metadata`. Packets route through the existing L1-L6 validator.

### 8. Filesystem-First, No Database
All source records, documents, indexes, tracker files, and reports live as files on disk. Copy the `knowledge/` directory to any machine and the full knowledge subsystem state transfers with it.


## Trust Tier Model

| Tier | Meaning | Auto-Escalate? | Confidence Contribution |
|------|---------|----------------|------------------------|
| `verified` | Confirmed by operator or trusted authority | Never | 0.9 |
| `reputable` | From a known-reliable source | Never | 0.7 |
| `unknown` | No provenance to establish trust | Never | 0.4 |
| `suspicious` | Indicators of bias or low quality | Never | 0.2 |
| `rejected` | Explicitly rejected by operator | Never | 0.0 |

Trust tiers inform routing and confidence scoring but **never become authoritative truth automatically**.

## Governance Boundaries (Non-Negotiable)

1. No silent source trust escalation
2. No source deletion (stale/archived only)
3. No hidden provenance rewriting
4. No unverifiable synthesized claims
5. All generated conclusions must reference source lineage
6. Contradictory sources must remain visible (both sides preserved)
7. Stale sources must remain recoverable
8. No autonomous web crawling or browser control
9. No embeddings or vector storage yet
10. No autonomous knowledge synthesis loops

## What Was NOT Implemented (by Design)

- No PDF ingestion or OCR
- No embeddings / vector search
- No RAG pipelines
- No autonomous web crawling
- No browser-based content extraction
- No LLM-generated memory promotion
- No social media ingestion
- No background indexing daemons
- No live sync systems
- No cloud storage

## Future v2.5+ Extension Points

1. **PDF Ingestion** — `ingest_pdf()` using pymupdf/marker-pdf for text extraction, with page-level provenance
2. **Embedding Compatibility** — Optional embedding index alongside keyword index, with provenance-tracked vector generation
3. **PLUR Adapter** — Bridge to the PLUR planning/learning/unlearning/reasoning framework for research pipeline orchestration
4. **CryER Integration** — Wire knowledge sources into CryER signal extraction workflows, enabling research-informed recon
5. **Web Ingestion Gateway** — Operator-auditable single-page ingestion (not crawling), with trust classification on every fetch
6. **Confidence Calibration** — Track actual vs claimed confidence across packets to tune the trust model
7. **Source Diff History** — Track content changes across re-ingestion of updated sources with diff provenance
