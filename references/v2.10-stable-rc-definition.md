# OverCR v2.10.0 — Stable Release Candidate Definition

## Scope

v2.10.0 is a **stabilization-first** release. Its purpose is to prove
the OverCR v2 substrate is stable, installable, reproducible,
understandable, recoverable, and operationally trustworthy.

## What's Included

- **release/** package (7 modules): SemanticCompatibility, InstallValidator,
  ReleaseBuilder, ReleaseManifest, VersionMatrix, ReproducibilityChecker,
  OperatorReadiness
- 4 new validation scripts under scripts/
- Complete version lineage tracking (v1.0.0 → v2.10.0)
- Machine-readable release manifest
- Clean release archive builder (tar.gz + SHA256 + metadata)
- Operator readiness validation (docs, demos, governance)

## What's NOT Included (scope freeze)

- No new subagents, workflow primitives, execution authority
- No new sandbox capabilities or web ingestion powers
- No autonomous behavior, distributed orchestration, browser automation
- No cloud dependencies, new runtime models, or major architectural systems
- No automatic migrations, telemetry, analytics, auto-update systems

## Stable Candidate Qualification

A release qualifies as **stable candidate** when:
1. Full test suite passes (29/29 tests, 85s)
2. All validation scripts exit 0
3. Clean extraction validates without errors
4. Semantic compatibility shows no breaking changes
5. Release archive builds cleanly (no bytecode, no runtime debris)
6. Operator docs are complete and demos are runnable
7. Version matrix shows full backward compatibility

## What Remains for a True Stable Release

- Long-running soak tests (multi-hour execution)
- Cross-platform validation (Linux bare metal, macOS)
- Operator acceptance testing with real workflows
- Security audit of sandbox isolation
- Performance benchmarking under load
