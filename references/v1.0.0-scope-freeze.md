## Purpose

This document is the definitive inventory of what is IN and what is OUT for
the v1.0.0 stabilization release. Once Phase 1 is complete, no item may move
from OUT to IN without a documented exception and scope revision.

The freeze covers:
- Code (all `.py` in `runtime/`, `tools/`, `tests/`)
- Config schemas (all `.yaml` in `config/`)
- Packet type registry (all valid `packet_type` values)
- Subagent surface area (all supported subagents)
- Task lifecycle (12-state machine)
- Workflow primitives (DAG model, policy, runner)
- Test inventory (26 regression tests in test_manifest.json)
- Reference documentation (all `.md` in `references/`, `docs/`)

### 9. Stable Substrate Guarantees

The following guarantees are invariant in v1.0.0:

| Guarantee | What It Means | Enforcement |
|-----------|---------------|-------------|
| Filesystem truth is authoritative | Chat is ephemeral; disk is canonical | `task_store.py`, `audit_writer.py` |
| Approval required for outbound | No outbound action without operator approval | `approval_gate.py` L4+L5 |
| Sovereignty enforced | Subagents never call each other directly | `workflow_graph.py`, `workflow_policy.py` |
| Audit append-only | No deletion or modification of audit entries | `audit_writer.py` write-only |
| Failed workers safe | Timeout/crash leaves task in inspectable state | `worker_runner.py` |
| Deterministic replay | State can be reconstructed from filesystem | `replay_runner.py` |
| No autonomous outbound | Substrate never makes network requests | Runtime boundary — no network stack |
| No autonomous fs mutation | Workers cannot write files outside workspace | Worker stdout-only contract |
| Governance bypass impossible | No operator or model can skip validation | L1-L6 pipeline, no override hooks |
| Cold-start recoverable | New instance boots from filesystem alone | `boot.sh`, `soul.md`, `replay_runner.py` |

### 10. Unstable / Experimental Areas

These exist in the codebase but are **NOT stable guarantees** for v1.0.0.

| Area | Why Unstable | Commitment |
|------|------------|------------|
| Model/provider names in configs | Data, not API | May change as providers evolve |
| Open WebUI integration | No native binding exists | Out of scope; may be added in v1.x |
| Workload examples (CRM, outreach) | Applications, not substrate | Examples only; not guaranteed stable |
| Demo workflows in `examples/` | Illustrative | May be replaced; workflow PRIMITIVES are stable |
| `scripts/package_release.sh` | Tooling | May change format; substrate unaffected |
| `scripts/check_release_clean.py` | Tooling | May gain new checks; substrate unaffected |
| Path-specific model configs | Environment-specific | Not part of substrate contract |

These areas may evolve without a major version bump because they are NOT part
of the substrate surface area.
