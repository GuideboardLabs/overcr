# OverCR v2.7.0 — Sandbox Backend Governance

## Selection Rules

1. Policy runs first — always. Backend cannot override policy.
2. `backend_preference="auto"`: tries firejail → bubblewrap → local
3. `backend_preference="firejail"`: firejail first, local fallback
4. `backend_preference="bubblewrap"`: bubblewrap first, local fallback
5. `backend_preference="local"`: local only, no fallback attempt
6. `fallback_allowed=false`: if preferred backend unavailable, execution is BLOCKED (not silently downgraded)
7. Every selection decision is recorded in `backend_selected` audit entry

## Remaining Attack Surfaces (Honest Update v2.7)

1. **Read-only system file access**: `cat /etc/hostname` succeeds because cat is read-only. Bubblewrap/Firejail backends mitigate via namespace isolation. Local backend still exposed.

2. **No seccomp on local backend**: Local backend has zero syscall filtering. Bwrap and Firejail backends address this. v2.8: seccomp profiles for local.

3. **Environment leakage via /proc**: Empty `env={}` blocks inherited env, but `/proc/self/environ` accessible if not namespace-isolated. Bwrap hides proc; Firejail profiles can too.

4. **TOCTOU race**: Sub-millisecond window between snapshot and mutation. Single-threaded execution mitigates. Kernel isolation doesn't close this — it's inherent to any snapshot-before-write pattern.

5. **Binary argument edge cases**: Different parsers handle metacharacters differently. The arg scanner catches literal tokens only. Regex-based scanning for v2.8.
